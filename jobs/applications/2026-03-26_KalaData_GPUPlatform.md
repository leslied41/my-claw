## Job Description
Company: Kala DataAbout KalaKala Data is building energy-backed compute infrastructure by colocating modular data centres with low-cost power generation. We currently operate multiple sites running high-density compute workloads and are transitioning these into GPU-based infrastructure for AI and high-performance computing.Our core thesis is that energy is local, compute is global — and by controlling power, we can deliver materially lower-cost compute.Kala is a funded business with active deployments and expansion underway, providing a strong foundation to scale both infrastructure and platform capabilities.The OpportunityWe are looking for an engineer to build the next stage of our platform — a system that allows customers to access GPU compute through Kala, while dynamically provisioning capacity across both Kala-owned infrastructure and third-party providers.This is not a typical SaaS role. You will be building a thin orchestration layer over distributed GPU supply, solving real-world infrastructure problems across energy, compute, and networking.What You'll Be BuildingYou will lead the development of a platform that:Accepts customer requests for GPU compute (via API and/or simple UI)Routes workloads to available Kala-owned GPU infrastructureDynamically provisions external GPU capacity when local supply is constrained (e.g. via third-party APIs)Abstracts away infrastructure differences so users experience a single, unified platformTracks usage, performance, and cost across multiple compute providersOver time, this system will evolve into a fully integrated GPU platform combining owned and third-party capacity.Key ResponsibilitiesDesign and build the core job routing and scheduling layerIntegrate with external GPU providers via API (provisioning, monitoring, teardown)Build a clean abstraction layer for compute resourcesDevelop APIs for users to request and manage workloadsImplement monitoring, logging, and cost tracking across infrastructureWork closely with the founders on product direction and system designWhat We're Looking ForStrong backend engineering experience (Python, Go, or similar)Experience building APIs and distributed systemsFamiliarity with cloud infrastructure (AWS, GCP, or similar)Experience with containers (Docker) and ideally orchestration (Kubernetes)Comfortable working with infrastructure-level problems (networking, compute, resource allocation)Ability to operate independently and make architectural decisionsNice to have:Experience with GPU workloads or ML infrastructureExperience with job schedulers (e.g. Slurm, Kubernetes jobs)Familiarity with marketplace or multi-provider systemsWhy This RoleYou will be building the core system that underpins the businessDirect exposure to energy + compute infrastructure (not just cloud abstraction)Opportunity to shape architecture from day oneHigh ownership and autonomyCompensationCompetitive salary + equityFlexible depending on experience
