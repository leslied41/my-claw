## Job Description
The role
As a Software Engineer in one of our talented teams, you'll play a key role in delivering high‑quality internal and external services by designing, developing, supporting and continuously improving software solutions used by APNIC staff and members across the Asia‑Pacific region.

You'll work across the full software development lifecycle, collaborate closely with other engineers, and contribute to a strong culture of code quality, documentation, and continuous improvement.

What you'll be doing
- Maintain and enhance existing software, including bug fixes and feature improvements
- Own assigned work items end‑to‑end, from analysis through to deployment
- Actively participate in code reviews to improve solution quality and consistency
- Follow configuration and release management practices, including producing clear technical documentation
- Manage source control activities such as branching, merging and versioning in line with team standards
- Ensure clear, consistent communication across all activities, including documentation where required
- Adhere to Software Development processes and support others in following them
- Contribute across all phases of the software development lifecycle: Requirements, design, development, testing, deployment and maintenance

About you
You're a motivated Software Engineer who enjoys solving problems, writing clean and maintainable code, and working collaboratively in a team environment.

You'll bring:
- Bachelor of Information Technology, Computer Science, or equivalent practical experience
- 2–5 years of relevant professional software development experience
- Strong communication skills, with the ability to explain technical concepts clearly
- A proactive, problem‑solving mindset and attention to detail

Technical skills and experience
You have working knowledge or hands‑on experience in:
- Java and core Java libraries
- RESTful web services
- Database design and implementation
- Linux or other Unix‑like operating systems
- Source control management, such as Git
- User interface design principles and information architecture

Desirable experience
The following experience would be highly regarded:
- Ability to prioritise competing requirements and work systematically to deliver outcomes
- Proficiency in one or more Asia‑Pacific regional languages
- Experience developing software in Perl 5, ideally using object‑oriented design
- Experience working in a member‑based or not‑for‑profit organisation
- Familiarity with technologies such as: Spring, Docker and Kubernetes, System administration on Linux or Unix‑like platforms, TCP/IP addressing structures and networking concepts
